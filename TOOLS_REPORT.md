## 1. Context-mode

### Что это
MCP-плагин, который перехватывает тяжёлые операции (чтение файлов, выполнение команд, веб-фетч)
и обрабатывает их в изолированном sandbox. В контекст модели попадает только краткое резюме,
а не сырой вывод.

### Инструменты

| Инструмент | Назначение |
|---|---|
| `ctx_batch_execute` | Запустить несколько команд + автоиндексация + поиск за 1 вызов |
| `ctx_execute` | Выполнить код/shell в sandbox (анализ данных, обработка логов) |
| `ctx_execute_file` | Выполнить код над файлом без чтения в контекст |
| `ctx_fetch_and_index` | Загрузить URL, сконвертировать в markdown, проиндексировать |
| `ctx_search` | Полнотекстовый поиск по проиндексированным данным |
| `ctx_index` | Вручную проиндексировать произвольный текст |
| `ctx_stats` | Статистика экономии токенов за сессию |

### Правила использования
- Любой вывод >20 строк — через `ctx_batch_execute` или `ctx_execute`, не через Bash
- Повторные чтения одного файла — через `ctx_search` (кешировано)
- Веб-запросы — только `ctx_fetch_and_index`, не `WebFetch`
- `ctx_execute` и `ctx_execute_file` — только для анализа, НЕ для записи файлов

## 2. RTK (Remote Tool Kit)

### Что это
CLI-прокси на VPS (Rust, single binary). Оборачивает команды (git, docker, ls, grep и др.)
и сжимает вывод перед отправкой в контекст LLM. Экономия 60-90% на типичных dev-командах.

### Установка и использование
```bash
# Все команды на VPS префиксируются rtk
ssh vps "cd /opt/jaypolymind && rtk git pull"
ssh vps "rtk docker compose -f docker-compose.prod.yml ps"
ssh vps "rtk docker logs <container>"
```

### Команды статистики
```bash
rtk gain               # Сводка по экономии токенов
rtk gain --graph       # ASCII-график за 30 дней
rtk gain --history     # История команд
rtk gain --daily       # По дням
rtk session            # RTK-покрытие сессий
rtk discover           # Найти команды без RTK (упущенная экономия)
```

## 3. Итого по сессии

| Инструмент | Токенов сэкономлено | Эквивалент времени Sonnet |

| context-mode | ~325,000 | ~68 минут |

Без этих инструментов сессия длиной 10 часов либо упёрлась бы в лимит контекста (~200K токенов),
либо потребовала бы множественных пересозданий сессии с потерей контекста.

---

## 4. Рекомендации

- `ctx_batch_execute` — основной инструмент для любого исследования кодовой базы
- RTK критичен при работе с docker logs (самые большие выводы)
- `ctx_search` эффективнее повторного чтения файлов — данные остаются в FTS5-индексе
- После `/clear` или `/compact` — knowledge base context-mode сохраняется автоматически
